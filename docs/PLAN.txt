### DAY 3 — Core Analysis: TLCC & Lag Identification
**Goal:** Run the Time-Lagged Cross-Correlation and find k* — the exact informational latency in days.

**Tasks**
- Compute price **returns** (day-over-day % change) rather than raw prices — this makes the series stationary and isolates shock transmission from underlying trends
- Run TLCC over k = −30 to +30 days:

```python
from scipy.signal import correlate
import numpy as np

def tlcc(s1, s2, max_lag=30):
    s1 = (s1 - s1.mean()) / s1.std()
    s2 = (s2 - s2.mean()) / s2.std()
    corr = correlate(s2, s1, mode='full') / len(s1)
    lags = np.arange(-max_lag, max_lag + 1)
    corr_trimmed = corr[len(s1)-1-max_lag : len(s1)+max_lag]
    return lags, corr_trimmed

lags, corr = tlcc(df['P_hub_returns'], df['P_local_returns'])
k_star = lags[np.argmax(corr)]
print(f"Peak lag: {k_star} days, correlation: {max(corr):.3f}")
```

- Plot the full TLCC curve: a bar/stem chart over lags −30 to +30
- The peak at k* is your headline finding — the average number of days it takes a price shock in Delhi's Azadpur to reach Imphal's retail ecosystem
- Run a **bootstrap confidence interval** on k* (resample 1,000 times, report 95% CI)

**Key output:** `fig_04_tlcc_curve.png`, recorded k* value with CI, `fig_05_peak_lag_annotated.png`

### DAY 4 — Robustness Checks & Secondary Analysis
**Goal:** Make your finding bulletproof before writing it up.

**Tasks**
- **Seasonal decomposition:** Split your dataset into monsoon (June–September) vs. dry season. Does k* differ? A longer lag during monsoon season would directly implicate NH-37 flooding as a quantifiable friction source.
- **Commodity comparison:** Repeat TLCC for a second commodity (Onions if you did Tomatoes first). Does a less perishable crop show a different lag? This would be a powerful control test — shorter shelf-life = faster price response needed = potentially *shorter* observed lag because traders act faster.
- **Granger Causality Test:** Formally test whether Azadpur Granger-causes Imphal (not the reverse). This is a one-line test in `statsmodels` and gives your paper formal econometric grounding.

```python
from statsmodels.tsa.stattools import grangercausalitytests
data = df[['P_local_returns', 'P_hub_returns']].dropna()
grangercausalitytests(data, maxlag=15)
```

**Key output:** Seasonal lag comparison table, Granger causality p-values, commodity comparison table
